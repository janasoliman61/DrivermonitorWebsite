Quick start

  1. Start the Python model server (if you use it): run the Flask server in flask_server/ as described in README.
  2. Start the Node server: npm install, then npm start in project root.
  3. Open http://localhost:5000 in your browser. Click "Start Camera".

UI controls

  • Start Camera — asks for permission and starts capturing frames.
  • Stop Camera — stops capture and camera stream.
  • Interval — adjust how often frames are captured and sent (ms).

What to expect

When frames are sent, the backend will return JSON results from the model(s). Results appear on the right side as alerts and a small log of events.

Integrating your models

If your models are Python-based, put them in flask_server/models/ and update flask_server/app.py to load and run them. If models run in JS, you can also run them in Node or the browser.

Key points:

  • Load models once at server start (not per request).
  • Preprocess images to the right size and normalization your model expects.
  • Return a simple JSON object the frontend can display (see README schema).

Troubleshooting

  • Camera permission errors: serve the page from http://localhost or HTTPS.
  • Blank video: ensure the browser supports getUserMedia and no other tab is using the camera exclusively.
  • Slow inference: increase the capture interval or run models on a faster machine/GPU.

Next steps

  • Consider WebSocket for lower latency streaming.
